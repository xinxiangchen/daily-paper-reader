## 摘要
机器人操作是一项复杂的任务，需要视觉理解、物理推理、规划和闭环控制。通用基础模型（FM）在这些能力中部分已变得非常强大，尤其是视觉和推理。为了将其应用于通用机器人策略，当前的方法通常通过微调机器人数据来将现有的FM转换为视觉-语言-动作（VLA）模型，以输出低级动作。然而，由于微调可用的数据和计算资源有限，VLA通常比前沿FM小数个数量级，这反过来限制了它们的通用能力。受FM通过视觉界面操作软件的能力日益增强的启发，我们探究了相同的能力是否足以控制机器人。我们提出了VIA（用于机器人控制的视觉界面智能体），一个将机器人控制重新定义为代理任务的框架：一个由现成FM驱动的智能体通过基于浏览器的3D界面操控机械臂，通过截取屏幕截图、发出直观命令、观察结果并进行调整。该智能体未接受任何针对机器人的微调，也无法访问特权状态信息：它感知视觉输入，并通过少量通用工具进行操作。VIA继承了智能体的通用推理、闭环错误恢复、以及根据观察进行规划和重新规划的能力。它在零样本情况下解决了多种桌面操作任务，同时适用于Claude Code和Codex。使用最强模型（Fable 5），它在三个LIBERO-Goal任务上达到了96.7%的成功率，并在一个长期跨度的彩虹组装任务上达到100%的成功率。性能随着底层模型的规模和强度而提高。这些结果表明，前沿智能体已经具备了在适当接口下直接迁移到机器人控制的技能：从某种意义上说，你的编码或计算机使用智能体秘密地也是一个机器人控制智能体。

## Abstract
Robot manipulation is a complex task that requires visual understanding, physical reasoning, planning, and closed-loop control. General-purpose foundation models (FMs) have grown remarkably capable of some of these, especially vision and reasoning. To leverage this for generalist robot policies, current methods typically involve converting existing FMs into vision-language-action (VLA) models by fine-tuning on robot data to output low-level actions. However, VLAs are often orders of magnitude smaller than frontier FMs given the limited data and compute available for fine-tuning, which in turn limits their general capability. Inspired by the growing ability of FMs to operate software through visual interfaces, we ask whether that same competence suffices to control a robot. We present VIA (Visual Interface Agent for robot control), a framework that recasts robot control as an agentic task: an off-the-shelf FM-powered agent drives a manipulator through a browser-based 3D interface by taking screenshots, issuing intuitive commands, observing the outcome, and adjusting. The agent receives no robot-specific fine-tuning and no access to privileged state information: it perceives visual input and acts through a small set of general tools. VIA inherits the agent's general reasoning, closed-loop error recovery, and ability to plan and re-plan from what it observes. It solves a diverse suite of tabletop manipulation tasks zero-shot with both Claude Code and Codex. With the strongest model (Fable 5) it achieves 96.7% success on three LIBERO-Goal tasks and 100% on a long-horizon rainbow assembly task. Performance improves with the scale and strength of the underlying model. These results suggest that frontier agents already possess skills that transfer directly to robot control given the right interface: your coding or computer-use agent is, in a sense, secretly a robot-control agent.

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：VIA将机器人控制转化为一个基于浏览器的3D界面上的人机交互任务。智能体通过截取界面截图观察环境，然后调用一组通用的MCP（Model Context Protocol）工具来设置虚拟目标夹爪（blue target gripper）的位姿（位置+朝向+开/闭），最后执行一个简单的比例积分控制器（PI controller）将真实夹爪移动到目标位姿。整个过程是闭环的：智能体观察→思考→执行→再观察，可随时调整计划。
- **关键技术细节**：
  - **视觉界面**：基于RGB-D相机重建3D点云作为主工作区，同时显示第三人称和腕部相机实时画面。智能体仅通过截图感知，不能访问非视觉数据（如关节角度）。
  - **MCP工具集**：分为四类：
    - 观察工具：`screenshot`（截图）、`hover`（悬停定位坐标）、`gripper_get_pose`（获取当前目标夹爪位姿）、`gripper_show_rotation_gizmo`（显示旋转辅助圆环）、`camera_get_pose`（获取相机位姿）。
    - 动作工具：`gripper_teleport_via_click`（点击瞬移）、`gripper_drag`（拖拽）、`gripper_translate`（精确平移）、`gripper_advance_or_retreat`（沿接近轴前进/后退）、`gripper_rotate`（绕局部轴旋转）、`gripper_toggle`（开/合夹爪）、`gripper_reset`（重置目标夹爪到实际位姿）。
    - 相机工具：`camera_orbit_via_key`、`camera_pan_via_key`、`camera_zoom`、`camera_reset`。
    - 执行工具：`execute_waypoint`（将当前目标夹爪位姿发送给控制器，控制真实机器人运动）、`end_episode`（结束回合）。
  - **工具设计原则**：极简主义（尽可能接近人类操作）和智能体人因工程（针对离散截图观察设计，避免连续拖拽等难以理解的操作）。
- **算法流程**（智能体循环）：
  1. **观察**：调用`screenshot`获取当前界面截图。
  2. **思考**：基于任务目标、当前状态、历史观察，决定下一步操作。
  3. **操作**：调用一个或多个工具调整目标夹爪位姿（如`gripper_teleport_via_click`、`gripper_rotate`、`gripper_translate`等），每次调用后检查返回的截图确认效果。
  4. **执行**：当目标夹爪位姿满意时，调用`execute_waypoint`，控制器将真实夹爪插值移动到该位姿。
  5. **重复**：检查结果，若任务未完成则回到步骤1，否则调用`end_episode`终止。
- **无需微调**：智能体本身（Claude Code或Codex）未做任何针对机器人的微调，仅使用一个短小的系统提示（约650词）说明界面基本概念和高效操作技巧。任务提示仅包含目标描述，甚至零样本即可工作。

## 6. 论文的主要结论与发现

1. **零样本机器人控制可行**：VIA在最小提示下即可零样本解决多种桌面操作任务，整体成功率CC-Opus 70%、CC-Fable 88%、Codex-5.5 60%、Codex-5.6-Sol 62%。这表明通用智能体在适当接口下已具备直接用于机器人控制的感知、推理和工具使用能力。
2. **性能随模型规模提升**：在同系列中，更强模型（Fable 5 vs Opus 4.8；GPT-5.6-Sol vs GPT-5.5）通常带来更高成功率或更低调用次数。CC-Fable在LIBERO-Goal任务上达到96.7%，彩虹组装100%。
3. **文本演示可显著提升性能**：提供给CC-Opus示例航点列表后，LIBERO-Goal任务成功率从77%提升至100%，说明VIA支持从文本反馈学习，具有学习与反思的潜力。
4. **成本可控**：成功回合的API成本大多在几美元以内，Codex更便宜（平均$3-7），在实际部署中有一定可行性。
5. **当前限制**：依赖最前沿大模型，推理慢且贵；不能用于动态任务；高精度任务（如T-block）仍困难，放置误差容忍度仅约8mm。

## 7. 优点

- **方法创新**：将机器人控制重新定义为计算机使用任务，彻底避免了机器人数据收集和微调，使机器人控制直接受益于基础模型和计算机使用智能体的飞速发展。
- **接口设计巧妙**：构建的3D点云界面类似于3D设计软件（如Blender），利用了模型可能在训练中见过的类似视觉环境；MCP工具集简短、直观且满足智能体观察习惯（离散、反馈式）。
- **闭环与错误恢复**：智能体每次操作后观察结果，可随时调整计划，具有天然的稳健性，避免了开环执行中的累积误差。
- **扩展性**：框架支持自动工具改进（智能体可反馈修改工具）、反思学习（通过文本演示改进），以及未来可用于数据收集或自动重置。
- **实验设计全面**：包含多种任务类型、多个模型和提示级别，定量评估了成功率、工具调用效率、成本等，数据比较完整。
- **结果令人兴奋**：96.7%的LIBERO-Goal零样本成功率和100%的长时彩虹组装任务成功率，展示了前沿智能体在机器人控制上的巨大潜力。

## 8. 不足与局限

- **仅在模拟器中验证**：所有实验在robosuite模拟器中完成，未在真实机器人上运行，真实环境中的传感器噪声、物理不一致性等挑战可能明显降低性能。
- **仅使用最强大模型，成本与延迟高**：当前依赖Claude Code或Codex的顶级模型，推理速度慢（每步需要截图传输和模型推理），不适于高速或动态任务（如接球）。API成本虽低但连续运行仍偏高。
- **精度不足**：对高精度任务（如T-block，容许误差仅8mm）表现不佳（成功率仅10-40%），说明当前视觉理解和工具精度尚不足以替代专用控制方法。
- **实验对比不充分**：未在同一设置下与VLA（如RT-2、OpenVLA）或CaP（如Code-as-Policies、ProgPrompt）进行直接对比，无法量化VIA相对于现有方法的优劣。
- **任务局限**：任务均为准静态桌面操作，未涉及复杂环境（如动态障碍物、可变形物体、多步协作等）。系统提示中明确不提供任何感知辅助API（如分割、检测），完全依赖模型的像素级理解，这在复杂场景下可能不够可靠。
- **评估细节不足**：每个配置仅10个种子，未报告方差；Rainbow任务成功判定主观；未进行消融实验（如不同工具集、不同视图组合的影响）。
- **依赖于现成界面和开源模拟器**：校准的RGB-D相机、点云重建、指定位姿插值控制器等都可能成为实际部署的瓶颈。目前框架未处理碰撞避免（智能体可能规划出碰撞路径）。
- **缺乏错误与失败模式分析**：论文未深入分析失败案例的原因（例如何时抓取过高、何时规划失败、何时工具调用错误），这些信息对于改进框架至关重要。
