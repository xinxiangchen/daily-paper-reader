## 摘要
扩散策略通过迭代去噪动作块实现强大的视觉运动控制，但重复去噪使得实时部署的计算需求很高。基于缓存的方法通过重用中间激活来降低推理成本，但现有的免训练调度通常均匀地在各块之间分配计算，忽略了块之间的异构冗余性，导致性能与效率的权衡欠佳。为弥补这一差距，我们引入了演化缓存调度（EVO），这是一个免训练的加速框架，通过演化搜索全局地制定缓存刷新调度。EVO将每个候选表示为一个覆盖块-时间步栅格的完整调度。这样，迭代去噪过程中的冗余Transformer计算可以通过缓存重用跳过，同时保持闭环展开性能。为了使搜索实用化，EVO引入了冗余感知初始化（用有前景的调度初始化种群）和以目标为条件的早停（在达到期望性能目标时验证并终止）。离线优化的调度可以直接插入预训练的扩散策略中，无需重新训练。大量操作基准测试表明，EVO在保持接近完整性能的同时大幅减少计算，实现了高达8.05倍的动作生成加速，并将FLOPs从15.77G降至低至1.96G。源代码可在https://github.com/pillom/EVO获取。

## Abstract
Diffusion policies achieve strong visuomotor control by iteratively denoising action chunks, but repeated denoising makes real-time deployment computationally demanding. Cache-based methods reduce inference cost by reusing intermediate activations, but existing training-free schedules typically allocate computation uniformly across blocks, ignoring heterogeneous redundancy across blocks and leading to a suboptimal performance-efficiency trade-off. To bridge this gap, we introduce Evolving Cache Schedules (EVO), a training-free acceleration framework that globally schedules cache refreshes via evolutionary search. EVO represents each candidate as a complete schedule over the block-timestep lattice. Thus, redundant transformer computations during iterative denoising can be skipped through cache reuse while preserving closed-loop rollout performance. To make the search practical, EVO introduces redundancy-aware initialization, which seeds the population with promising schedules, and target-conditioned early stopping, which verifies and terminates once a desired performance target is reached. The offline-optimized schedule can be directly plugged into pretrained diffusion policies without retraining. Extensive manipulation benchmarks show that EVO preserves near-full performance while substantially reducing computation, achieving up to 8.05x action-generation speedup and reducing FLOPs from 15.77G to as low as 1.96G. Source code is available at https://github.com/pillom/EVO.

# 论文总结：Evolving Cache Schedules for Fast Diffusion Policy Inference

## 1. 论文的核心问题与整体含义（研究动机和背景）

扩散策略（Diffusion Policy）通过迭代去噪过程模拟多模态动作分布，在机器人视触觉控制中展现出强大能力，但迭代推理带来的高计算延迟限制了其在实时、高频率控制场景中的部署。现有基于缓存的加速方法（如 EfficientVLA、BAC）通常采用固定间隔或按块等量分配计算预算的调度策略，忽略了不同 Transformer 模块（自注意力、交叉注意力、前馈网络）在去噪步进间存在显著异构冗余这一事实。作者通过实验发现，块之间的时序特征相似度分布和整体冗余程度差异很大（图1），因此均匀分配计算导致性能-效率权衡欠佳。为此，论文提出一种免训练的全局缓存调度框架 EVO，将调度问题转化为在“块×时间步”格点上的子集选择问题，通过进化搜索直接优化闭环展开成功率的全局缓存刷新计划，实现大幅加速且保持近全精度。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
将加速视为固定预算的缓存调度问题：给定 `B` 个可缓存模块（层内自注意力、交叉注意力、前馈网络，共 3×L 个单元）和 `T` 个去噪步，总预算 `K`，选择 `K` 个格子位置 `(b, t)` 执行原始残差计算并刷新缓存，其余位置重用最近缓存的残差。目标是最大化闭环任务成功率。

### 关键技术细节
- **调度表示**：每个候选调度 `S` 为 `K` 个不重复的 `(b, t)` 坐标集合。
- **适应度函数**：直接使用加速后的策略在 `N_eval` 个 rollout 上的平均成功率 `F(S)`。
- **进化搜索**：
  - 锦标赛选择、集合级交叉和变异、修复算子（去重、补位）、精英保留。
  - 利用规范签名缓存已评估调度的结果，避免重复 rollout。
- **冗余感知初始化**：在少量 uncached rollout 上收集各位置残差激活，计算余弦相异度 `d_{b,t}`，归一化后作为初始种群中部分个体的采样权重，偏置到低冗余（高相异）位置。同时保留随机个体以保持探索。
- **目标条件早停**：快速评估（20 episodes）触发后，用独立种子进行正式评估（50 episodes），若正式成功率不低于无缓存基线的 `δ_acc`（允许性能下降），则终止搜索。此举减少无效 rollout 开销。
- **离线优化后部署**：搜索一次性完成，固定最优调度后直接插入预训练策略，不改变任何网络参数、采样器或接口，运行时按公式（1）决定是否刷新缓存。

- 基线策略：预训练 Transformer 扩散策略（DP-T），全精度 15.77G FLOPs。
- 对比方法：

  - **BAC**：按块自适应调度，但预算在各块间固定。
- 配置：缓存预算设置为平均每模块更新步数 `M=8` 或 `M=10`，对应总格子数 `K=192` 或 `240`（24个模块×每个模块刷新M次）。

### 消融实验变体
- Block-wise GA：每块刷新步数固定相同。
- Block-Step GA：全局分配不限定块内预算。
- Block-wise GA + RI / Block-Step GA + RI：分别加入冗余感知初始化（RI），后者即完整 EVO。

## 6. 论文的主要结论与发现

- EVO 在保持与全精度模型相近平均成功率（PH: 0.78 vs 0.80; MH: 0.79 vs 0.79; 多阶段: 0.99 vs 0.99）的同时，实现 **8.05× 动作生成加速**，FLOPs 从 15.77G 降至约 1.96G–2.34G。推理频率从 2.23 Hz 提升至 7.57 Hz。
- 相比于 EfficientVLA，EVO 在复杂任务（Transport、Tool、Kitchen）上性能崩溃更少；相比于 BAC，EVO 以更低计算量达到相当或更好的平均成功率。
- **全局块-时间步预算分配**优于每块等量分配（Block-Step GA 优于 Block-wise GA）。
- **冗余感知初始化**显著提升搜索效率和最终性能（RI 带来 0.04~0.05 的绝对成功率提升，并减少 GPU-days 和 Search Steps）。
- 余弦相异度作为初始化度量效果最佳；保留随机个体有助于种群多样性。

## 7. 优点

- **免训练**：无需修改预训练策略、采样器或动作接口，即插即用。
- **直接优化任务性能**：不同于使用特征相似度等代理目标，EVO 用闭环成功率作为适应度，对齐最终部署目标。
- **全局化调度视角**：将预算在整个块-去躁步格点上自由分配，充分挖掘异构冗余。
- **高效搜索设计**：冗余感知初始化 + 目标条件早停显著降低离线搜索成本（平均 0.96 GPU-days）。
- **实验扎实**：多基准、多种子、多指标，消融全面，运行时分析详实，附录提供大量细节（调度表示、搜索轨迹等）。

## 8. 不足与局限

- **离线搜索仍需可观计算**：即使优化后，每任务仍需要约 1 GPU 天和数百次 rollout 评估，对于真实机器人直接部署可能仍然昂贵。论文承认这一点，但强调是一次性开销且可被后续多次部署摊销。
- **环境分布不变假设**：搜索基于同分布仿真环境，若迁移到不同环境或分布偏移（如光照、布局变化），调度可能需重新搜索。
- **未对比在线自适应调度**：EVO 产生固定离线调度，而 BAC 虽固定但采用局部自适应规则。若环境动态变化，固定调度可能不是最优。
- **实验仅限 Transformer 扩散策略**：结论可能不直接推广到 U-Net 或其他架构。
- **无真实机器人部署**：所有实验在仿真中完成，未在真实机器人上验证延迟和性能。
- **早停阈值 δ_acc 依赖人为设定**：论文中未探讨 δ_acc 敏感性，但实现中可能需要对基线性能有先验知识。
- **硬件限制**：仅使用单张 RTX 4090D GPU，未测试多 GPU 或不同平台的扩展性。
