## 摘要
长视界机器人规划需要同时推理语义任务结构和几何可行性。为了成功执行一项任务，机器人必须分解目标、选择任务相关对象并排序动作，同时确保规划满足空间约束，例如有限自由空间和物体碰撞。在这项工作中，我们提出了APIVOT，一种基于VLM的规划器，它自适应地交错语言和视觉思维以实现长视界规划。APIVOT学习利用语言进行语义推理，同时使用视觉思维作为想象的未来状态来内部验证几何可行性。在长视界厨房任务中，APIVOT优于通用VLM和先前的规划框架，在空间受限场景中取得了最大的提升。我们发现APIVOT学习了有意义的模态选择行为，证明了视觉-语言思维的自适应交错提高了规划成功率和推理效率。

## Abstract
Long-horizon robot planning requires jointly reasoning over semantic task structure and geometric feasibility. To successfully execute a task, a robot must decompose goals, select task-relevant objects, and sequence actions, while ensuring that plans satisfy spatial constraints such as limited free space and object collisions. In this work, we propose APIVOT, a VLM-based planner that adaptively interleaves language and visual thoughts for long-horizon planning. APIVOT learns to leverage language for semantic reasoning, while using visual thoughts as imagined future states for internal verification of geometric feasibility. On long-horizon kitchen tasks, APIVOT outperforms general-purpose VLMs and prior planning frameworks, achieving the largest gains in spatially constrained settings. We find that APIVOT learns meaningful modality selection behavior, demonstrating that adaptive interleaving of vision-language thoughts improves both planning success and reasoning efficiency.

### 1. 论文的核心问题与整体含义（研究动机和背景）

*   **核心问题**：长期机器人规划需要同时处理**语义任务结构**（如任务分解、物体选择）和**几何可行性**（如空间约束、物体碰撞）。现有的基于大语言模型（LLM）或视觉语言模型（VLM）的规划器通常能生成语义上合理的行动序列，但在需要精细几何推理的场景（如判断物体是否能放入有限空间）中表现不佳。
*   **研究背景**：现有方法往往通过外部模块（如运动规划器、可行性检查器）来提供几何反馈，但这些反馈是事后修正，并未融入规划器自身的内部推理过程。作者认为，一个高效的规划器应该能够在内部自主交错使用语言和视觉模态进行推理，以适应任务对语义和几何不同侧重的需求。语言擅长语义推理，而视觉表示能高效地编码空间布局信息。

### 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想**：APIVOT 是一个基于 VLM 的规划器，它学习在推理轨迹中自适应地交错使用**语言思考**和**视觉思考**。语言用于处理语义决策，而视觉思考则通过**想象未来状态**来内部验证几何可行性。
*   **关键技术细节**：
    1.  **多模态推理轨迹**：规划器首先生成一个包含交错语言和视觉符号的推理轨迹，然后再输出具体动作序列。
        *   视觉思考（Visual Thought）：在轨迹中以固定长度的特殊视觉 token 序列表示（例如 `<|image_start|><|image_pad|>^K<|image_end|>`），其解码器隐藏状态编码了未来状态的视觉内容，后续 token 可以通过注意力机制访问这个预测的未来状态表示。
    2.  **三阶段训练课程**：这是方法论的核心，通过逐步训练教会模型理解、生成和自适应使用视觉思考。
        *   **阶段一：视觉理解**。模型学习**使用**提供的视觉思考来指导规划。在这个阶段，真实的视觉特征被注入到视觉 token 的位置，模型只需学习如何基于这些状态生成正确的推理和动作。
        *   **阶段二：视觉生成**。模型学习**生成**视觉思考。要求模型自己生成视觉 token 及其对应的潜在视觉状态，并通过余弦相似度损失将其与真实未来图像的编码特征对齐。
        *   **阶段三：自适应模态选择**。模型学习**决定何时**使用视觉思考。训练数据中的视觉思考只在几何约束较大的步骤中出现。模型通过此阶段的训练，学会仅在对规划有益时才生成视觉思考，从而提高效率。
    3.  **参考输出生成**：从成功演示中构造带有标注的参考轨迹，为每个子目标标注约束类型（符号前提、当前几何约束、未来几何可行性）和模态标签，然后使用大语言模型将其扩展为自然的推理轨迹。

### 7. 优点：方法或实验设计上的亮点

*   **方法创新**：将几何推理**内化**到规划器的推理轨迹中，而非依赖外部模块，这是核心亮点。三阶段的课程学习设计清晰、合理，循序渐进地教会模型一项复杂技能。
*   **自适应计算**：提出的自适应模态选择机制是实际部署中的关键。它根据任务步的需求动态分配计算资源（视觉思考是高成本的），而不是无差别地使用，实现了性能与效率的双赢。

*   **可解释性**：通过分析统计不同约束下的视觉思考使用率，让模型的决策过程变得可解释，展示了模型学习到的内在逻辑。

### 8. 不足与局限

*   **仿真环境局限**：所有实验均在 **KitchenWorlds** 模拟器中进行。虽然任务设计覆盖了复杂的空间规划挑战，但真实世界中的视觉多样性、物理参数不确定性、传感器噪声等可能带来更大的挑战，向真实世界迁移是一个重要但尚未验证的步骤。
*   **视觉表示泛化性**：视觉思考的潜在表示是在特定任务分布下训练的，可能无法完全捕捉真实世界中几何和外观的全部变化。在更复杂的真实场景中，可能会失效。
*   **自适应学习的偏差**：自适应模态选择是通过监督微调学习的，其学习目标（即参考轨迹中的模态标签）是由**模拟器导出的启发式规则（阈值）** 确定的。这可能导致模型学习到的选择策略是近似的，未能完全优化任务成功率。作者也提到，使用强化学习来优化这个选择可能带来更好的效果。
*   **单模态推理**：框架仅使用文本和图像两种模态。作者指出，引入更丰富的表示（如点云、3D边界框）可能会使模型在更多抽象层次上进行推理，进一步提升性能。
