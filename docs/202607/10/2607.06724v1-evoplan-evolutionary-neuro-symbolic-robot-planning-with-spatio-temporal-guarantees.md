## 摘要
基于大语言模型的机器人规划器虽然流畅，但无法保证其计划的可执行性或安全性。经典的PDDL规划器可以保证这些属性，但前提是问题被完全指定，并且它们未能充分利用大语言模型阅读上下文和修复计划的能力。本文提出了一种由三部分组成的神经符号框架。所有大语言模型调用均使用本地托管的开源权重模型，因此整个流程可部署在机器人上，无需依赖云端。首先，一个离线程序从演示数据中挖掘单一的全局信号时序逻辑约束。该程序恢复编码规则（例如，从nuPlan驾驶日志中挖掘的在红灯前停车）或群体偏好（例如，从SCAND遥操作中挖掘的社交导航舒适度），具体取决于演示所编码的内容。由于演示是单类信号，我们通过反事实扰动和大语言模型违规生成器生成缺失的负例，然后通过进化搜索拟合约束。我们使用挖掘的约束来屏蔽Bench2Drive上的视觉语言驾驶策略和HA-VLN-CE上的两个离散动作导航策略。其次，一个进化PDDL规划器：大语言模型提出并修复计划，程序化验证器决定哪些计划存活，计划中已验证的部分在迭代中增长。我们在开放世界ALFWorld文本基准上测试该规划器，它击败了强基线，并且在目标词汇与动作模型词汇不匹配时保持鲁棒性。第三，一个约束执行循环：将规划器的计划编译为路径点，根据挖掘的约束检查路径点，并在违反约束时重新规划。我们通过使用Gazebo模拟器的演示来展示整个流程。

## Abstract
LLM-based robot planners are fluent but cannot guarantee that their plans are executable or safe. Classical PDDL planners can guarantee these properties, but only after the problem is fully specified, and they make poor use of an LLM's ability to read context and repair plans. This paper presents a neuro-symbolic framework with three parts. All LLM calls use a locally-hosted open-weight model, so the pipeline can be deployed on-robot with no cloud dependency. First, an offline procedure that mines a single global Signal Temporal Logic (STL) constraint on mobility from demonstration data. The procedure recovers codified rules (e.g., stopping at red lights, mined from nuPlan driving logs) or population preferences (e.g., social-navigation comfort, mined from SCAND teleoperation), depending on what the demonstrations encode. Because the demonstrations are a one-class signal, we generate the missing negatives with counterfactual perturbations and an LLM violation generator and then fit the constraint by evolutionary search. We use the mined constraint to shield a vision-language driving policy on Bench2Drive and two discrete-action navigation policies on HA-VLN-CE. Second, an evolutionary PDDL planner: an LLM proposes and repairs plans, programmatic validators decide which ones survive, and the validated portion of the plan grows over iterations. We test the planner on the open-world ALFWorld Text benchmark, where it beats strong baselines and stays robust when the goal vocabulary does not match the action-model vocabulary. Third, a constrained execution loop: the planner's plan is compiled into waypoints, the waypoints are checked against the mined constraint, and the planner re-plans on a violation. We illustrate the full pipeline via demonstrations using the Gazebo simulator.

### 论文提出的方法论

- **核心思想**：采用“生成-验证-筛选”的演化计算模式。LLM负责生成和修复计划，而程序化验证器（符号验证器、时空逻辑约束）决定哪些计划是安全且可执行的。整个过程分为三个核心组件：

    1.  **离线STL约束挖掘 (Offline STL Constraint Mining)**：
        *   **目标**：从无标注的“好”的演示数据中，自动学习一个单一的、全局的、适用于所有移动动作的时空安全约束 $\Phi_{mob}$。
        *   **技术细节**：这是一个从单类数据（只有正样本）中学习二分类边界的过程。
            *   **反事实负样本生成**：对演示轨迹进行十种程序化扰动（如超速、逆行、闯红灯、贴人行走等），并利用LLM生成真实感更强的违规样本，构造正样本集 $D^+$ 和负样本集 $D^-$。
            *   **进化STL拟合**：采用LLM引导的进化搜索（类似OpenEvolve），在预定义的STL语法空间（包含速度、距离、时间差等信号）中搜索一个公式，该公式能最大化正样本的鲁棒性（$\rho$），同时最大化负样本的惩罚（违反约束）。
            *   **约束特性**：生成的 $\Phi_{mob}$ 是全局的，不绑定于特定任务或动作，适用于任何计划中的移动行为。它可以是从数据中挖掘出的“红灯停”等代码规则，也可以是“保持社交距离”等群体偏好。

    2.  **进化PDDL规划器 (Evolutionary PDDL Planner)**：
        *   **目标**：从一个PDDL问题$P$和领域$D$出发，生成一个符号上有效的计划 $\pi$。
        *   **算法流程**：
            *   初始化一个候选计划，然后进入一个迭代循环。
            *   每一代（iteration），LLM基于上一代的验证反馈（validated prefix），对当前最佳计划提出编辑（如插入缺失前提的动作、修复对象绑定）。
            *   新产生的候选计划通过一个四级验证级联（PDDL语法检查、VAL模拟、目标可达性、STL鲁棒性预测）。无效计划被降级，有效计划保留。
            *   **关键特性**：已验证的前缀长度会在迭代中单调增长（如图2所示），直至整个计划符号有效。
            *   **开放世界与词汇不匹配**：该算法通过`belief_to_problem` 处理未见过的对象，并利用一个LLM调用进行谓词符号映射（`map_predicates`）来解决目标词汇与动作模型词汇不一致的问题。

    3.  **约束执行循环 (Constrained Execution Loop)**：
        *   **目标**：将PDDL计划安全地部署到物理世界。
        *   **流程**：PDDL规划器生成计划 $\pi = (a_1, a_2, ..., a_n)$。对于每个动作 $a_k$，导航层将其展开为一条具体的路径点序列 $w_k$。在执行前，$w_k$ 会接受 $\Phi_{mob}$ 的检查。
        *   **决策逻辑**：如果 $w_k$ 满足 $\Phi_{mob}$，则执行；如果违反，则拒绝该动作，提交已执行的已验证前缀 $a_1...a_{k-1}$，更新世界状态，并**重新调用规划器**生成一个新的计划来替换剩余部分。这构成了一个“规划-检查-重新规划”的闭环。

1.  **单一全局STL约束可行且有效**：通过反事实样本和进化搜索，可以从单类演示数据中学习到一个通用的、适用于所有动作的时空安全约束。该约束能很好地泛化到未见任务，作为盾牌可以大幅减少违规行为（如闯红灯、碰撞），同时不显著牺牲任务成功率。
