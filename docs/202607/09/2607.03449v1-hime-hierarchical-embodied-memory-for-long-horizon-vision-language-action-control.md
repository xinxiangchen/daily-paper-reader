## 摘要
当前的视觉-语言-动作（VLA）模型在机器人操作方面表现出色，但由于依赖即时观测，它们往往难以处理需要长期记忆和推理的非马尔可夫任务。现有解决方案面临“频率-能力悖论”：更强的推理模型速度太慢，无法用于实时控制，而更快的模型则缺乏足够的推理能力。为了解决这种架构上的错配，我们提出了HiMe，一种分层具身记忆框架，将具身智能解耦为用于执行的高频执行器、用于工作记忆的哨兵以及用于长期策略的规划器。我们还引入了一种基于跨模态语义模式和主动管理机制的动态知识系统，使机器人能够通过“添加、更新和删除”操作保持记忆可塑性。这种分层设计有效平衡了实时执行与慢速思考规划之间的冲突，显著提高了长时域任务的成功率。实验表明，该方法不仅优于扁平记忆基线，还展现出基于人类偏好自我修正内部知识的新能力。

## Abstract
Current Vision-Language-Action (VLA) models excel at robotic manipulation but often struggle with non-Markovian tasks requiring long-term memory and reasoning due to their reliance on immediate observations. Existing solutions face a ''frequency-competence paradox,'' where stronger reasoning models are too slow for real-time control, while faster models lack sufficient reasoning capabilities. To resolve this architectural misalignment, we propose HiMe, a Hierarchical Embodied Memory framework that decouples embodied intelligence into a high-frequency Executor for execution, a Sentry for working memory, and a Planner for long-term strategy. We also introduce a dynamic knowledge system based on cross-modal semantic schemas and active management mechanisms, allowing robots to maintain memory plasticity through ''Add, Update, and Delete'' operations. This hierarchical design effectively balances the conflict between real-time execution and slow thinking planning, significantly improving success rates in long-horizon tasks. Experiments demonstrate that this approach not only outperforms flat memory baselines but also exhibits the novel ability to self-correct its internal knowledge based on human preferences.

### 1. 核心问题与整体含义（研究动机和背景）
- **背景**：现有视觉-语言-动作（VLA）模型在大规模预训练下表现出色，但大多假设环境瞬态可观测（马尔可夫性质），即策略 $p(a_t|o_t,l)$ 仅依赖当前观测 $o_t$ 和指令 $l$。然而，实际长时域操作任务往往是非马尔可夫的，最优动作依赖于历史事件序列或隐藏在传感器流之外的潜在信息。
- **核心问题**：存在“频率-能力悖论”——更强的推理模型（如大型VLM）速度过慢，无法满足实时控制的高频低延迟要求；而速度足够快的模型推理能力有限，零样本泛化能力弱。现有方法（如训练增强记忆或引入大型VLM作为记忆容器）要么上下文窗口有限、难以优化长程依赖，要么因规模约束失去实时性。
- **研究意义**：解决这一矛盾是实现自主、自适应机器人长期操作的关键，HiMe通过分层记忆架构模拟人类认知的多存储结构，旨在平衡实时执行与深度推理。

### 2. 方法论：核心思想、关键技术细节
#### 核心思想
- **分层架构**：将具身智能解耦为三个功能层，对应不同时间分辨率的记忆：
  - **执行器（Executor）**：高频VLA模型，管理瞬态记忆（当前观测 $o_t$），专注于感知-运动协调，保证物理稳定性。
  - **哨兵（Sentry）**：轻量级VLM，管理工作记忆（滑动窗口内的最近观测 $\mathcal{W}_t = \{o_{t-h_s},...,o_t\}$），异步过滤连续感知流，识别关键状态转换（如子任务完成/失效）。
  - **规划器（Planner）**：重量级VLM，管理情景记忆（长时记忆），仅在哨兵触发的离散时刻被调用，进行深层策略推理与记忆更新。
- **动态知识系统**：记忆不是被动累积，而是基于跨模态语义模式（视觉+文本）和主动管理机制（Add、Update、Delete）的动态知识库，维持记忆可塑性与一致性。

#### 关键技术细节
- **问题形式化**：长时域操作建模为部分可观测下的序贯决策，目标生成动作序列 $a_{0:T}$ 满足指令 $l$。最优动作依赖完整历史 $h_t = (o_0,a_0,...,o_t)$，但连续处理 $h_t$ 计算不可行，因此采用分层策略。
- **记忆组件**：
  - 瞬态记忆 $\mathcal{T}_t$：执行器管理，仅含当前观测。
  - 工作记忆 $\mathcal{W}_t$：哨兵管理，滑动窗口 $h_s=8$ 帧，用于子目标验证与故障检测。
  - 情景记忆 $\mathcal{E}_t$：规划器管理，包括：
    - 上下文记忆 $\mathcal{E}_t^c$：多模态键-值存储，累积任务相关信息（图像+文本描述），支持通过文本键检索。
    - 程序记忆 $\mathcal{E}_t^p$：结构化子任务计划列表，每个子任务带状态标签（active/pending/done）。
- **分层策略分解**：
  - 执行器：$a_t \sim \pi_e(a_t | o_t, \tau_t)$，其中 $\tau_t$ 是当前活跃子目标，保证无状态、高效实时控制。
  - 哨兵：每 $n_m=5$ 步评估一次，输出触发信号 $u_t$，若子目标完成或无效则触发规划器：
    $$u_t = \begin{cases} \mathbb{I}[\pi_s(\mathcal{W}_t, \tau_t) > \delta], & t \equiv 0 \pmod{n_m} \\ 0, & \text{otherwise} \end{cases}$$
  - 规划器：被触发后执行：
    1. **上下文检索**：将指令和最近观测编码为查询 $q_t$，从情景记忆中检索最相关的条目 $\mathcal{M}_{\text{ret}} = \text{TopK}(\mathcal{E}_t^c, q_t)$。
    2. **记忆更新与重规划**：基于指令 $l$、工作记忆 $\mathcal{W}_t$、检索上下文 $\mathcal{M}_{\text{ret}}$ 和当前程序计划 $\mathcal{E}_t^p$，联合更新上下文记忆（Add/Update/Delete）和程序记忆，输出新计划，选择下一个子目标。
- **实现细节**：
  - 规划器：GPT-4o（主要）或 Qwen3-VL-30B（开源替代）。
  - 哨兵：Qwen3-VL-8B。
  - 执行器：基于 $\pi_{0.5}$ 微调（任务特定数据），训练参数：30k步，批量大小256，峰值学习率5e-5，余弦退火。
  - 记忆后端：向量数据库（OpenAI text-embedding-3）+ 余弦相似度检索。

### 6. 主要结论与发现
- HiMe在所有任务上平均Task Progress达90%，显著优于所有基线（Transient Memory 26%、Flat Memory 65%等），接近人类上限（94%）。
- **哨兵的价值**：通过强制时序一致性（减少任务切换）和提供高质量工作记忆（均匀采样8帧而非最近帧），显著提升性能（如Transient Memory从14%提升到26%+）。
- **跨模态记忆最优**：图像保留空间/识别细节，文本保留语义/逻辑，两者结合性能最佳（平均90%）。
- **主动管理优于被动存储**：FIFO（68%）因早期信息被驱逐而变差；无管理（86%）虽保存全部但引入冗余噪声；HiMe通过Add/Update/Delete保持一致性达到90%。
- **效率优势**：HiMe相比Flat Memory将API调用减少约3倍（如Object Search从5.4降至1.8），因为哨兵将规划器从每步调用变为子任务级稀疏调用。
- **自我修正能力**：在用户偏好变化时，HiMe能更新甚至删除过期记忆，表现出新颖的“自我修正”能力。

### 7. 优点
- **创新点**：系统性地解耦记忆为三层（瞬态/工作/情景），并首次将主动记忆管理（Add/Update/Delete）引入具身VLA，解决频率-能力悖论。
- **实用性强**：哨兵作为轻量级触发器，大幅降低重量级VLM的调用频率（~3x减少），在真实机器人上实现了低延迟部署。
- **消融全面**：从模态、管理机制、容量、规划器替换等多维度验证每项设计的必要性。
- **跨模态记忆**：将视觉和文本紧密结合，既保证空间定位又支持语义推理。
- **开源友好**：提供了开源规划器Qwen3-VL-30B的完整实验，改善可重复性。

### 8. 不足与局限
- **实验覆盖**：
  - 仅在真实机器人上评估（WidowX-250），缺乏标准仿真benchmark，限制了受控比较和基准级复现。
  - 任务主要基于简单操作原语（pick-and-place等），未涉及更复杂的灵巧操作或移动操作。
  - 扩展性实验仅到3轮（12个子任务），未充分表征极长时域（>20轮）或更大环境下的表现。
- **哨兵局限性**：Recall较低（约35%），存在“漏报”风险（错过完成信号），虽通过固定时间回退缓解，但可能引发无限循环或过度执行。
- **依赖外部API**：主实验使用GPT-4o（封闭API），延迟较高（P50~38s），本地部署虽降低延迟但增加了硬件成本。
- **偏差风险**：训练数据量小（60个演示），可能对特定场景过拟合；未讨论在不同机械臂或完全新任务上的泛化性。
- **应用限制**：当前框架假设子任务可被明确分解，对于连续、无明确阶段边界的任务可能不适用；记忆管理算法依赖VLM的语义理解，在复杂歧义场景下可能出错。
